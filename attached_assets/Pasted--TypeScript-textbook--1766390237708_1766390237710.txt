あなたはTypeScriptに強いフロントエンドエンジニアです。現在の「簿記教科書」ページ（/textbook）にある 簿記の一連の流れ「取引→仕訳→勘定記入→決算書作成」 の各カードをクリック可能にし、クリックすると モーダル（Dialog） が開いて、そのステップの説明とサンプル（仕訳表／帳簿／決算書など）が見られる機能を追加してください。

重要条件
	•	既存UI（iPhoneっぽいシンプルさ、余白多め、カード風）を壊さない
	•	クリック対象は4カード（取引/仕訳/勘定記入/決算書作成）。矢印部分はクリック不要でもOK
	•	実装は “データ駆動” にする（モーダル内容をハードコードで分岐しない）
	•	モーダルは閉じやすく：×ボタン、ESC、背景クリックで閉じる（可能なら）
	•	アクセシビリティ：フォーカストラップ、aria-label、キーボード操作
	•	レスポンシブ：スマホ幅ではモーダル内容が縦スクロールで読める

UI仕様
	•	モーダル上部：タイトル（例「仕訳（Journal Entry）」）
	•	次に短い説明（2〜4行）
	•	「IN / OUT」ブロック（既存の教科書カードのIN/OUTと同様の見せ方）
	•	その下に「サンプル」セクション（タブ or アコーディオンでOK）
	•	取引：領収書/請求書/契約書などの“例”（ダミーの紙面カード）
	•	仕訳：仕訳表（借方/貸方、科目、金額）のテーブル
	•	勘定記入：総勘定元帳（T勘定 or 仕訳から転記された明細の簡易表）
	•	決算書：損益計算書（P/L）と貸借対照表（B/S）の簡易サンプル（表でOK）
	•	モーダル下部：次に進む導線ボタン
	•	「このステップを練習する」→ 該当モードへ遷移（仮のリンクでもOK）

データ設計（必須）
	•	flowSteps.ts（または flowSteps.json）を作成して、以下の構造で4ステップ分定義する

type FlowSample =
| { type: “table”; title: string; columns: string[]; rows: (string | number)[][] }
| { type: “cards”; title: string; items: { label: string; value: string }[] }
| { type: “statement”; title: string; blocks: { heading: string; lines: { label: string; amount: number }[] }[] };

type FlowStep = {
id: “transaction” | “journal” | “ledger” | “statements”;
title: string;
subtitle?: string;
description: string; // 2〜4行
inLabel: string[];
outLabel: string[];
keyPoint?: string; // 注意点など1〜2行
samples: FlowSample[];
};
	•	教科書ページ側は flowSteps をmapしてカードを描画し、クリックでモーダルに渡す

サンプルデータ（MVPに入れて）
	1.	取引（transaction）

	•	IN: 領収書、請求書、契約書
	•	OUT: 仕訳の検討
	•	samples(cards): 「取引の例」(日付/内容/金額/証憑)

	2.	仕訳（journal）

	•	IN: 取引内容
	•	OUT: 仕訳（借方/貸方）
	•	samples(table): 仕訳表
例）「備品を現金30,000円で購入」
借方：備品 30,000 / 貸方：現金 30,000

	3.	勘定記入（ledger）

	•	IN: 仕訳
	•	OUT: 総勘定元帳（科目別の増減と残高）
	•	samples(table): 仕訳→転記後の元帳例（備品勘定、現金勘定）

	4.	決算書作成（statements）

	•	IN: 元帳残高（試算表）
	•	OUT: 損益計算書、貸借対照表
	•	samples(statement): P/L（売上、費用、当期純利益）と B/S（資産、負債、純資産）の簡易例

実装アウトプット
	•	既存ページの該当箇所にクリックハンドラ追加
	•	StepModal（Dialog）コンポーネント実装
	•	FlowSampleRenderer を作り sample.type ごとに描画
	•	flowSteps.ts の追加
	•	最後に、変更したファイル一覧と動作確認手順を提示

この仕様で、クリック→モーダルで説明＋サンプル閲覧ができるように実装してください。スタック（Next.js/Reactなど）は既存リポジトリに合わせて判断し、最小差分で入れてください。
